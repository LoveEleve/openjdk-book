引入与前文已有系统功能重叠的新组件时，必须第一时间解释两者关系，不能孤立讲解。

**规则：** 假设读者记住了前面章节的内容——当新系统出现时，先用对比讲清"这个和前面的 XX 是什么关系、为什么两个都要有"，再展开新系统本身的细节。

**Why:** 用户读到 LogConfiguration::initialize 时立刻联想到 3.3.3 节的 ostream_init/defaultStream::instance，质疑"这里感觉和之前的 ostream_init 有重叠？"——说明读者在主动跨章节关联。如果书中不主动解释，读者会困惑"为什么要两套输出系统？"。

**How to apply:** 遇到以下模式时立即用对比表格：
- 两个输出系统（tty vs UL）
- 两个初始化阶段有重叠职责
- 同一个概念在不同层次有不同实现

对比表格至少包含：创建时机、创建对象、用途、输出目标。
