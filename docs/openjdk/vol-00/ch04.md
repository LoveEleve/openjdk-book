# jdk11u-copy：裁剪、CMake 与 IDE

上一章的 Make 构建教会了你如何从源码编译 JDK。但这不够——你改一行 HotSpot 源码后，需要等 1 分 31 秒才能验证。`jdk11u-copy` 把这个循环缩短到 10 秒。

本章讲清楚这个项目做了什么——它不是一个简单的副本，而是从 jdk11u 基础上系统性重构出的成果：先裁出一个服务器专用 JDK，再给它配上 CMake 构建系统，最后用 CLion 实现秒级增量开发和源码跳转。

### 核心问题：改了 HotSpot 源码，怎么最快验证

[此处插入：Make 需要完整 1m31s → CMake 只需 10s。因为 .so 是通过 dlopen 动态加载的，替换文件后 JVM 启动时自然加载新版，不需要重新链接启动器、不需要重新生成 JNI 头文件、不需要重新编译 Java 类]
