# 第10章：universe_init —— JVM 运行时地基的入口

> `init_globals()` 第 120 行调用 `universe_init()`。函数名意为"宇宙初始化"——它创建 Java 堆、Metaspace、类加载器元数据、符号表，把 JVM 从"有线程没业务"推进到"能跑字节码"的关键一步。本章从最基础的知识点出发，逐层往上建。

`universe_init()` 定义在 `universe.cpp:675`，75 行代码。它是 `init_globals()` 调用的第 10 个子函数，也是第一个有返回值的——之前的 9 个函数都是 void。它之后的 `interpreter_init`、`javaClasses_init`、`compileBroker_init` 都依赖它在堆和 Metaspace 上建立的基础设施。

它不是一个大函数，但它是 "Genesis"——JVM 运行时世界的第一个实质性的地基。

## 返回值和依赖

`universe_init()` 返回 `jint`——JNI_OK (0) 成功，JNI_EINVAL/JNI_ERR 失败。失败时 `init_globals` 的返回值被 `create_vm` 检查——失败意味着 JVM 无法启动。

依赖关系（重头戏在后续章节）：
- `initialize_heap` 创建堆 → ch11
- `Metaspace::global_initialize` 创建元空间 → ch12
- `MetaspaceShared::initialize_shared_spaces` 加载 CDS → ch13
- SymbolTable/StringTable/ResolvedMethodTable → ch14
- 6 个 LatestMethodCache → ch15

本章只覆盖 `universe_init` 中的辅助初始化——在"主函数"（堆/Metaspace）就绪后、"CDS/类加载"开始前的过渡阶段。
