# 第2章：JavaMain —— 跨进 JNI_CreateJavaVM

上一章结束在 `pthread_create(ThreadJavaMain)`，新线程的入口是 `ThreadJavaMain`，它只是签名的适配器——真正执行的是 `JavaMain(args)`。

`JavaMain` 是第一章所有准备工作的终点。到这里，JVM 还不存在，`ifn` 里的三个函数指针还只是三个地址。这一章就做一件事——调用 `ifn->CreateJavaVM(...)` 把 JVM 创建出来。

---

## JavaMain() 全貌

`JavaMain` 在 `java.c:413-578`，165 行。先看完整源码：

```c
/* === src/java.base/share/native/libjli/java.c === */

JavaMain(void *_args) {
    /* ═══════ 阶段1：参数解包 ═══════ */
    JavaMainArgs *args = (JavaMainArgs *) _args;
    int argc = args->argc;
    char **argv = args->argv;
    int mode = args->mode;
    char *what = args->what;
    InvocationFunctions ifn = args->ifn;

    JavaVM *vm = 0;
    JNIEnv *env = 0;
    jclass mainClass = NULL;
    jclass appClass = NULL;
    jmethodID mainID;
    jobjectArray mainArgs;
    int ret = 0;
    jlong start = 0, end = 0;

    /* ═══════ 阶段2：创建 JVM ═══════ */
    RegisterThread();
    start = CounterGet();
    if (!InitializeJVM(&vm, &env, &ifn)) {
        JLI_ReportErrorMessage(JVM_ERROR1);
        exit(1);
    }

    /* ═══════ 阶段3：特殊模式快速通道 ═══════ */
    if (showSettings != NULL) {
        ShowSettings(env, showSettings);
        CHECK_EXCEPTION_LEAVE(1);
    }
    if (showResolvedModules) {
        ShowResolvedModules(env);
        CHECK_EXCEPTION_LEAVE(1);
    }
    if (listModules) {
        ListModules(env);
        CHECK_EXCEPTION_LEAVE(1);
        LEAVE();
    }
    if (describeModule != NULL) {
        DescribeModule(env, describeModule);
        CHECK_EXCEPTION_LEAVE(1);
        LEAVE();
    }
    if (printVersion || showVersion) {
        PrintJavaVersion(env, showVersion);
        CHECK_EXCEPTION_LEAVE(0);
        if (printVersion) {
            LEAVE();
        }
    }
    if (validateModules) {
        LEAVE();
    }

    /* ═══════ 阶段4：没给参数 → 打印帮助 ═══════ */
    if (printXUsage || printUsage || what == 0 || mode == LM_UNKNOWN) {
        PrintUsage(env, printXUsage);
        CHECK_EXCEPTION_LEAVE(1);
        LEAVE();
    }

    FreeKnownVMs();

    /* ═══════ 阶段5：追踪日志 ═══════ */
    if (JLI_IsTraceLauncher()) {
        end = CounterGet();
        JLI_TraceLauncher("%ld micro seconds to InitializeJVM\n",
                          (long)(jint)Counter2Micros(end - start));
    }

    ret = 1;

    /* ═══════ 阶段6：加载主类 ═══════ */
    mainClass = LoadMainClass(env, mode, what);
    CHECK_EXCEPTION_NULL_LEAVE(mainClass);
    appClass = GetApplicationClass(env);
    NULL_CHECK_RETURN_VALUE(appClass, -1);

    /* ═══════ 阶段7：构建 String[] args ═══════ */
    mainArgs = CreateApplicationArgs(env, argv, argc);
    CHECK_EXCEPTION_NULL_LEAVE(mainArgs);

    /* ═══════ 阶段8：dry-run 快速通道 ═══════ */
    if (dryRun) {
        ret = 0;
        LEAVE();
    }

    /* ═══════ 阶段9：PostJVM 初始化 ═══════ */
    PostJVMInit(env, appClass, vm);
    CHECK_EXCEPTION_LEAVE(1);

    /* ═══════ 阶段10：查找并调用 main() ═══════ */
    mainID = (*env)->GetStaticMethodID(env, mainClass, "main",
                                       "([Ljava/lang/String;)V");
    CHECK_EXCEPTION_NULL_LEAVE(mainID);
    (*env)->CallStaticVoidMethod(env, mainClass, mainID, mainArgs);
    ret = (*env)->ExceptionOccurred(env) == NULL ? 0 : 1;
    LEAVE();
}
```

165 行，10 个阶段。分界线在第 2 阶段——`InitializeJVM` 返回前 JVM 不存在，返回后 JVM 已经活了。

## 阶段1：参数解包

第一章结尾的 `ContinueInNewThread` 把五个值打包进 `JavaMainArgs` 结构体，通过 `pthread_create` 的 `void* args` 参数传进来。这里解包回各自的变量：

```c
JavaMainArgs *args = (JavaMainArgs *) _args;
int argc = args->argc;
char **argv = args->argv;
int mode = args->mode;
char *what = args->what;
InvocationFunctions ifn = args->ifn;
```

`JavaMainArgs` 的定义：

```c
typedef struct {
    int    argc;
    char **argv;
    int    mode;
    char  *what;
    InvocationFunctions ifn;
} JavaMainArgs;
```

以 HelloWorld 为例，这些字段的值：

| 字段 | 值 | 说明 |
|------|-----|------|
| `argc` | `1` | 只剩 `"HelloWorld"` 一个参数（`-cp` 和 `/data/workspace/demo` 已被 `ParseArguments` 消费） |
| `argv` | `{"HelloWorld"}` | 剩余的非选项参数 |
| `mode` | `LM_CLASS` | 启动模式：通过类名启动 |
| `what` | `"HelloWorld"` | 要执行的类名 |
| `ifn` | `{CreateJavaVM, GetDefaultJavaVMInitArgs, GetCreatedJavaVMs}` | 三个函数指针，初始化为 0 |

`RegisterThread()` 在 Linux 上是空函数——只有一行注释 `stubbed out for windows and *nixes`，不执行任何操作。

## 阶段2：InitializeJVM —— 创建 JVM

这是本章的核心。`InitializeJVM` 在 `java.c:1524-1549`：

```c
InitializeJVM(JavaVM **pvm, JNIEnv **penv, InvocationFunctions *ifn) {
    JavaVMInitArgs args;
    jint r;

    memset(&args, 0, sizeof(args));
    args.version = JNI_VERSION_1_2;
    args.nOptions = numOptions;
    args.options = options;
    args.ignoreUnrecognized = JNI_FALSE;

    if (JLI_IsTraceLauncher()) {
        int i = 0;
        printf("JavaVM args:\n    ");
        printf("version 0x%08lx, ", (long) args.version);
        printf("ignoreUnrecognized is %s, ",
               args.ignoreUnrecognized ? "JNI_TRUE" : "JNI_FALSE");
        printf("nOptions is %ld\n", (long) args.nOptions);
        for (i = 0; i < numOptions; i++)
            printf("    option[%2d] = '%s'\n",
                   i, args.options[i].optionString);
    }

    r = ifn->CreateJavaVM(pvm, (void **) penv, &args);
    JLI_MemFree(options);
    return r == JNI_OK;
}
```

15 行，做三件事：

**1. 组装 JavaVMInitArgs。** JNI 规范定义了 `JavaVMInitArgs` 结构体——告诉 JVM 用什么版本启动、传哪些选项：

```c
memset(&args, 0, sizeof(args));
args.version = JNI_VERSION_1_2;       // 0x00010002，JNI 1.2 版本号
args.nOptions = numOptions;           // 启动选项的个数
args.options = options;               // 选项数组指针
args.ignoreUnrecognized = JNI_FALSE;  // 不忽略未知选项——错了就报错
```

`options` 是一个全局数组，由 `JLI_Launch` 的 Step 7-10 中通过 `AddOption()` 逐步填充。包括：
- `-D` 系统属性（`sun.java.command`、`sun.java.launcher` 等）
- 用户命令行上的 JVM 参数（`-Xmx256m` 等）
- 编译期参数（`jargv`，如果有）

`JNI_VERSION_1_2` 是历史遗留——JNI 1.2 及以上版本使用相同结构体，实际运行的是 JDK 11 的完整 JNI 实现。

**2. 追踪日志。** `JLI_IsTraceLauncher()` 在没开追踪时不进这个 `if`，日常等于空操作。

**3. 调用 CreateJavaVM。** 这是第一章 `dlsym` 解析出的第一个函数指针——`ifn->CreateJavaVM`。它在 `jni.cpp:4098` 指向 `JNI_CreateJavaVM`：

```c
_JNI_IMPORT_OR_EXPORT_ jint JNICALL JNI_CreateJavaVM(JavaVM **vm, void **penv, void *args) {
    jint result = JNI_ERR;
    result = JNI_CreateJavaVM_inner(vm, penv, args);
    return result;
}
```

`JNI_CreateJavaVM` 是 JVM 的入口。调用成功后：
- `*vm` 指向一个 `JavaVM*` 实例——代表一个可以创建线程、销毁 JVM 的虚拟机接口
- `*penv` 指向一个 `JNIEnv*` 实例——代表当前线程的 JNI 环境，后续 `GetStaticMethodID`、`CallStaticVoidMethod` 等 JNI 调用都通过它

`JNI_MemFree(options)` 释放启动选项数组——传完参数后不再需要。返回值 `r == JNI_OK`（`JNI_OK = 0`）表示创建成功。

`InitializeJVM` 返回后，JVM 已经是一个完整运行的虚拟机了。后续所有代码都是用 JNI 和它交互。

## 阶段3：特殊模式快速通道

`InitializeJVM` 之后有一串 `if`，对应各种 `java` 命令的特殊模式。它们的共同点：JVM 已经活了，做一件特定的事，然后退出。

| 命令行 | 标志 | 做什么 | 然后 |
|--------|------|--------|------|
| `java -XshowSettings` | `showSettings != NULL` | `ShowSettings(env)` 打印 JVM 配置 | 继续往下走 |
| `java --show-module-resolution` | `showResolvedModules` | `ShowResolvedModules(env)` 打印模块解析信息 | 继续往下走 |
| `java --list-modules` | `listModules` | `ListModules(env)` 列出所有可观察模块 | `LEAVE()` |
| `java --describe-module X` | `describeModule != NULL` | `DescribeModule(env, X)` 描述指定模块 | `LEAVE()` |
| `java -version` / `--version` | `printVersion \|\| showVersion` | `PrintJavaVersion(env)` 打印版本 | `LEAVE()` |
| `java --validate-modules` | `validateModules` | 什么都不做 | `LEAVE()` |
| `java -?` / `-help` / `-X` | `printXUsage \|\| printUsage \|\| what==0 \|\| mode==LM_UNKNOWN` | `PrintUsage(env)` 打印帮助 | `LEAVE()` |

这些模式都不走到 `LoadMainClass`，更不会 `CallStaticVoidMethod`。我们的 HelloWorld 不匹配任何一个，因此穿过这串 `if`，继续执行。

`CHECK_EXCEPTION_LEAVE(n)` 是一个宏——如果 JNI 环境里挂起了异常，设置返回码 `n` 并执行清理后返回。`LEAVE()` 也是宏——做清理工作（`DetachCurrentThread` + `DestroyJavaVM`）后返回。

## 本章总结

`JavaMain` 的 10 个阶段，核心是第 2 阶段：

```c
r = ifn->CreateJavaVM(pvm, (void**)penv, &args);
```

这行代码调用的是 `JNI_CreateJavaVM`——从 `jni.cpp` 开始，进入了 HotSpot 的内部初始化流程。

`InitializeJVM` 做的事：组装 `JavaVMInitArgs`，调用第一章 `dlsym` 拿到的函数指针，传入 `options`（启动参数数组）。返回后 JVM 存在于 `*vm` 和 `*penv` 两个指针中。

调用链：

```
main() → JLI_Launch() → JVMInit() → ContinueInNewThread()
       → pthread_create(ThreadJavaMain) → ThreadJavaMain()
       → JavaMain() → InitializeJVM() → ifn->CreateJavaVM()
       → JNI_CreateJavaVM()
```

下一章进入 `JNI_CreateJavaVM`——`Threads::create_vm()` → `init_globals()` → `universe_init()`，JVM 初始化的心脏地带。
